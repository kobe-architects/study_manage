### 3.2 学習系

#### GET `/study-resources/{studyResource}/quiz` — 出題取得

クエリパラメータ:

| パラメータ | 型 | 既定 | 説明 |
|------------|----|------|------|
| count | int | 20 | 問題数。0以下で全件 |
| quizType | string | choice | choice / input |
| sectionIds | csv | 全セクション | 対象セクションID |
| importances | csv | 指定なし | 重要度絞り込み |
| labels | csv | 指定なし | ラベル絞り込み |
| ordered | bool | false | sort_order順出題 |
| vocabularyIds | csv | 指定なし | 特定単語のみ（リトライ/復習） |

レスポンス: `data[]`。各要素は `{ vocabulary: VocabularyResource, choices?: [{meaning, isCorrect}] }`。`quizType=choice` のときのみ choices を付与。

#### POST `/vocabularies/{vocabulary}/attempt` — 回答記録

リクエスト（RecordAttemptRequest）: `{ is_correct: boolean, quiz_type: 'choice'|'input' }`
処理: VocabularyAttempt を作成し、SM-2で学習統計を更新。
レスポンス: `{ data: { id, isCorrect, quizType } }`

#### GET `/study-resources/{studyResource}/vocabularies/stats` — 進捗統計

レスポンス `data`: `{ totalWords, masteredCount, learningCount, newCount, overallAccuracy, dueForReview }`

#### GET `/study-resources/{studyResource}/vocabularies/incorrect` — 誤答抽出

クエリ: `since`（必須, date） / `until`（任意, date, since以降）。
処理: 期間内に is_correct=false の attempt を持つ単語を重複排除して取得。
レスポンス: `{ data: VocabularyResource[], meta: { total, since, until } }`

## 4. 出題選定ロジック（VocabularyService::getQuizWords）

出題単語は以下の優先順位で選定する（`count > 0` かつ通常モードの場合）。

1. **特定単語指定（vocabularyIds）**: 指定IDを教材配下に限定して取得しシャッフルして返す（リトライ/復習用）。以降の選定はスキップ。
2. **絞り込み**: sectionIds / importances / labels で母集合を限定。
3. **順番通り（ordered=true）**: sort_order順で（count件 or 全件）返す。
4. **全件（count ≤ 0）**: 母集合全件をシャッフルして返す。
5. **間隔反復配分（通常）**: 以下の3群から配分して選定し、最後にシャッフル。

   | 区分 | 条件 | 配分 |
   |------|------|------|
   | 復習対象（due） | learning_stats.next_review_at ≤ now、next_review_at昇順 | 60%（ceil） |
   | 新規（new） | learning_stats に記録なし、シャッフル | 30%（ceil） |
   | 苦手（hard） | next_review_at > now、ease_factor昇順 | 残り（count − due − new） |

   選定数が count に満たない場合は母集合の残りからシャッフルして補充する。

> 注: due と new が ceil のため合計が count を超えうるが、最終的に `unique()->take(count)` で丸める。

### 4.1 4択選択肢生成（generateChoices）

- 正解 = 対象単語の meaning。
- ダミー（3件）は **同一品詞** の単語の meaning を優先してランダム取得。
- 不足分は品詞を問わず（正解・既選択を除外して）ランダム補充。
- 正解＋ダミーを配列化し `shuffle()` して返す（各要素 `{meaning, isCorrect}`）。

## 5. SM-2アルゴリズム（VocabularyService::recordAttempt / applySM2）

回答記録時に attempt を作成し、`firstOrCreate` で統計を取得（初期値 ease_factor=2.50, interval_days=0, repetition_count=0）して更新する。

### 正解時

1. `correct_count++`、`repetition_count++`
2. interval_days を更新:
   - repetition_count == 1 → 1日
   - repetition_count == 2 → 6日
   - それ以外 → `round(interval_days × ease_factor)`
3. `ease_factor = min(3.0, ease_factor + 0.1)`

### 誤答時

1. `incorrect_count++`
2. `repetition_count = 0`、`interval_days = 1`
3. `ease_factor = max(1.3, ease_factor − 0.2)`

### 共通

- `last_attempted_at = now()`
- `next_review_at = now() + interval_days 日`

> 注: 標準SM-2の品質評価(0〜5)は用いず、正誤2値に簡略化した独自実装。

## 6. 進捗統計算出（VocabularyService::getProgress）

教材配下の全単語と自ユーザーの統計から算出する。

| 指標 | 算出 |
|------|------|
| totalWords | 教材配下の単語数 |
| masteredCount | repetition_count ≥ 3 かつ interval_days ≥ 7 の統計数 |
| learningCount | 統計件数 − masteredCount |
| newCount | totalWords − 統計件数 |
| dueForReview | next_review_at ≤ now の統計数 |
| overallAccuracy | Σcorrect_count / (Σcorrect_count + Σincorrect_count) × 100（小数1桁、0件時0） |

単語が0件のときは全指標0を返す。

### 7.3 画面・コンポーネント

#### VocabularyQuizPage（クイズ画面）

フェーズ: `settings → quiz → results`。

- 設定（QuizSettingsCard）で start を受け、ウォッチ未起動なら開始確認ダイアログを表示。
- `executeStart` で `startQuiz` 実行。出題0件なら警告。
- 表示中の問題は `displayedQuestion`（ローカルref）で管理し、storeのリアクティビティから隔離。`displayKey` を `:key` に使い再マウント制御。
- 回答（handleAnswer）→ `submitAnswer`。次へ（handleNext）→ `advanceQuiz`、完了で results へ。戻る（handlePrev）→ `goBackQuiz`。
- 習熟度/メモ更新（handleUpdateVocabulary）→ `update` ＋ `updateQuizVocabulary`。
- 結果から誤答リトライ（handleRetryIncorrect）→ 誤答IDで `startQuiz`（vocabularyIds指定）。
- 小テスト印刷（handlePrintTestSheet）→ 出題条件で単語取得し TestSheetPrint を表示。fill_spelling は例文なし単語を除外。

#### QuizSettingsCard（出題設定）

- props: sections / loading / printLoading / totalWords
- emits: `start(settings)` / `printTestSheet(settings, options)`
- セクション（複数チップ・全選択/解除）、重要度（0/1/2）、ラベル（easy/normal/hard）、クイズタイプ（choice/input）、問題数（スライダー5〜min(totalWords,50) または全件）、出題順序（random/sequential）。
- 印刷ダイアログ: テストタイプ（meaning/spelling/fill_spelling）・回答方式（free/choice）。
- セクション未選択時は開始無効。

#### ChoiceQuizCard（4択）

- props: question / previousAnswer
- emits: `answer(isCorrect, {selectedIndex})` / `next` / `update-vocabulary(id, {proficiency?, memo?})`
- 語（クリックでTTS）・品詞・重要度★・ラベル・習熟度を表示。「選択肢を表示」後にA/B/C/Dを提示。回答で赤/緑＋アイコン。「わからない」で誤答扱い。
- 回答後: 成否アラート、例文（対象語を太字強調・TTS）、和訳トグル、習熟度トグル（更新しない/高/中/低）、メモチップ（編集ダイアログ）、画像、例文説明ダイアログ。
- マウント時に語を自動読み上げ（previousAnswerがあれば抑制）。

#### InputQuizCard（入力）

- props/emits は ChoiceQuizCard と同様（answer の extra は `{inputAnswer}`）。
- 意味（日本語）を提示し英単語を入力（autofocus、Enter送信）。大文字小文字無視で採点。
- 回答後の例文・習熟度・メモ・画像表示は4択と共通。

#### QuizResultsCard（結果）

- props: score / incorrectWords。emits: `retryIncorrect` / `newQuiz`
- 「正答数/出題数」と正答率を色分け表示（80%↑緑 / 50–79%橙 / <50%赤）。誤答単語一覧（答えON/OFF）。

#### TestSheetPrint（小テスト印刷）

- props: words / resourceName / testType / testFormat。emits: `close`
- 出力範囲: すべて/問題のみ/回答用紙のみ（fill_spelling除く）/解答のみ。
- 問題用紙（3カラム、fill_spellingは1カラム）、回答用紙、解答の各シートを生成。choice形式は単語プールから4択を生成。fill_spellingは例文の対象語を `( ____ )` に置換。解答数は `; , 、 ； ，` で分割して計数。`window.print()` で印刷（CSSでUI非表示）。

#### フラッシュカード（VocabularyFlashcardPage / FlashcardSettingsCard / FlashcardDisplay）

- 設定: QuizSettingsベース＋ wordDuration（2〜10秒, 既定5）/ meaningDuration（2〜10秒, 既定3）。emits `start(settings, wordDuration, meaningDuration)`。
- 表示: 全画面、語→（TTS）→意味→フェードアウト→300ms→次へ。末尾で周回数++して継続。タブ非表示で自動一時停止、クリックで一時停止/再開。進捗「現在/総数（n周目）」＋バー。
- 開始時、ウォッチ未起動なら確認ダイアログ（quiz画面と同様）。

#### VocabularyReviewPage（復習）

- 期間選択（過去3日/7日/1ヵ月/任意）→ `incorrect` API で誤答抽出 → 一覧表示 → クイズ開始（vocabularyIds指定で startQuiz）または小テスト印刷。

#### VocabularyTable / VocabularyFormDialog（管理）

- Table: 一覧（#/語/意味/品詞/重要度/ラベル/習熟度/学習比率/操作）、検索（語・意味・例文・和訳）、絞り込み（セクション/ラベル/重要度/習熟度）、行展開で例文表示。emits `edit` / `delete`。
- FormDialog: 単語追加/編集。語・意味必須、品詞・重要度・ラベル・習熟度・メモ・例文系・画像。新規時はフォーム復元（useFormPersistence）。画像はアップロード/差替/削除を本文保存と分離。

### 7.4 活動ウォッチ連携（useStudyTimerStore）

- 学習開始時に `start({ source:'vocabulary', studyResourceId, vocabularyType:'quiz'|'flashcard' })` で計測開始。
- localStorage に永続化し、停止時に duration 等を含む TimerResult を返す（実行ログ化に利用）。

## 8. 主要シーケンス

### 8.1 クイズ実行

```
User → QuizSettingsCard: 条件設定→開始
QuizSettingsCard → VocabularyQuizPage: emit start(settings)
VocabularyQuizPage → (ウォッチ未起動なら) 開始確認ダイアログ
VocabularyQuizPage → Store.startQuiz → GET /quiz (出題＋4択)
Store ← API: quizQuestions
loop 各問題:
  User → Card: 回答
  Card → Page: emit answer
  Page → Store.submitAnswer → (非同期) POST /attempt → Service.recordAttempt → applySM2
  User → Card: 次へ → Store.advanceQuiz
end
Page → QuizResultsCard: 結果表示（誤答リトライ可）
```

### 8.2 回答記録とSM-2更新

```
POST /vocabularies/{id}/attempt {is_correct, quiz_type}
 → VocabularyAttempt.create
 → LearningStat.firstOrCreate
 → applySM2(stat, is_correct)
    正解: rep++/interval(1→6→×EF)/EF+0.1(≤3.0)
    誤答: rep=0/interval=1/EF-0.2(≥1.3)
    next_review_at = now + interval
 → return {id, isCorrect, quizType}
```

| 所有者不一致 | 403 abort |
| 画像未存在 | 404 abort |
| 回答記録送信失敗 | フロントでサイレント無視（クイズ進行を継続） |
| 出題0件 | フロントで警告スナックバー表示し開始しない |
| fill_spelling 例文なし | 対象除外、全除外時は警告 |
| API一般エラー | スナックバーでエラー表示 |

## 11. 既知の設計上の注意点

- SM-2配分の ceil により due+new が count を超えうるが `unique()->take(count)` で吸収。
- `getQuizWords` の vocabularyIds 指定時は絞り込み・配分を行わず指定単語をそのまま返す。
- 回答記録は非同期・失敗許容のため、ネットワーク不調時に統計が欠損しうる（学習体験優先の設計）。
- TTS音声・印刷はブラウザ依存。
