# 要件定義書 — 英単語クイズ機能

**KGI/KPI管理＋タスク管理 統合ツール（個人用） / 学習支援サブシステム**

- 文書バージョン: 1.0
- 作成日: 2026-06-29
- 対象範囲: 単語帳（Vocabulary）・英単語クイズ・フラッシュカード・復習・小テスト印刷
- 本書は現状実装（ソースコード）から逆生成した要件定義である。

---

## 1. 概要

本機能は、本ツールの「教材・講義（StudyResource）」配下に紐づく **単語帳（Vocabulary）** を対象に、英単語の暗記・定着を支援する学習サブシステムである。

教材 → セクション → 単語、という階層でデータを管理し、登録済みの単語に対して以下の学習モードを提供する。

- **クイズ（4択 / 入力）** — 出題・採点・回答記録
- **フラッシュカード** — 自動めくり表示による高速インプット
- **間違えた単語の復習** — 指定期間に誤答した単語の再出題
- **小テスト印刷** — 紙の小テスト（問題用紙・回答用紙・解答）出力
- **単語帳管理** — 単語のCRUD、画像添付、Excelインポート/エクスポート

学習結果は **間隔反復学習（SM-2アルゴリズム）** に基づいて記録され、復習タイミングの最適化と進捗の可視化に利用される。また、学習時間は本ツールの「活動ウォッチ（StudyTimer）」と連携し、努力量（実行ログ）として蓄積できる。

---

## 2. 背景・目的

### 背景

- 大学受験・英語学習などの長期目標において、英単語の暗記は反復が要であり、感覚的な学習では定着度が把握しづらい。
- 既存の単語帳アプリは本ツール（目標達成OS）と分断されており、学習時間や進捗が一元管理できない。

### 目的

- 教材単位で単語を体系的に管理し、複数モードで反復学習できるようにする。
- 間隔反復（SM-2）により「いま復習すべき単語」を自動で優先出題し、効率的に定着させる。
- 学習の進捗（習得 / 学習中 / 新規 / 復習期限到来 / 正答率）を定量的に可視化する。
- 学習時間を活動ウォッチ経由で実行ログ化し、本ツールの努力量可視化に統合する。
- 紙の小テストにも出力でき、デジタル/アナログ両面での学習を可能にする。

---

## 3. 対象ユーザー・利用範囲

- 対象ユーザー: 本人のみ（単一ユーザー前提だが、データは `user_id` で分離管理）
- 利用デバイス: Web（PC＋スマホ）、レスポンシブ対応
- 認証: 既存の認証基盤（Laravel Sanctum）配下。全APIは `auth:sanctum` ミドルウェアで保護され、教材の所有者（`user_id`）のみがアクセス可能。
- 技術スタック: Laravel 12（バックエンド） / Vue 3 + Vuetify 3 + TypeScript + Pinia（フロントエンド） / MySQL 8.0

## 4. 用語定義

| 用語 | 説明 |
|------|------|
| 教材（StudyResource） | 単語帳のルートとなる学習リソース。所有者（user_id）を持つ。 |
| セクション（Section） | 教材を分割する単位（章・範囲など）。単語はセクションに属する。 |
| 単語（Vocabulary） | 1つの英単語エントリ。語・意味・品詞・重要度・ラベル・習熟度・メモ・例文・画像を持つ。 |
| 回答記録（Attempt） | 1問への回答1件の記録。正誤・出題形式・回答日時を保持。 |
| 学習統計（LearningStat） | 単語×ユーザー単位の集約統計。SM-2のパラメータ（ease_factor / interval / repetition）と次回復習日を保持。 |
| クイズタイプ | 出題形式。`choice`（4択）/ `input`（英単語入力）。 |
| 重要度（importance） | 0=無印 / 1=★ / 2=★★。 |
| ラベル（label） | `easy`（易）/ `normal`（普）/ `hard`（難）。 |
| 習熟度（proficiency） | `high`（高）/ `medium`（中）/ `low`（低）/ 未設定。ユーザーの主観的な定着度。 |
| SM-2 | 間隔反復学習アルゴリズム。正誤に応じて次回復習間隔を調整する。 |
| 活動ウォッチ（StudyTimer） | 学習時間を計測するストップウォッチ機能。学習を実行ログ化する。 |

## 5. 機能要件

### 5.1 単語帳管理（FR-V）

| ID | 機能 | 内容 |
|----|------|------|
| FR-V-01 | 単語一覧表示 | セクション内の単語を `sort_order` 順に一覧表示する。 |
| FR-V-02 | 単語登録 | 語・意味（必須）、品詞・重要度・ラベル・習熟度・メモ・例文（任意）を登録する。 |
| FR-V-03 | 単語編集 | 既存単語の各項目を更新する。 |
| FR-V-04 | 単語削除 | 単語を削除する（添付画像も同時に物理削除）。 |
| FR-V-05 | 一括削除 | 教材配下の全単語を一括削除する。 |
| FR-V-06 | 画像添付 | 単語に画像を1枚アップロード/差し替え/削除する（jpeg/png/gif/webp、最大5MB）。 |
| FR-V-07 | 検索・絞り込み | 語・意味・例文・和訳の全文検索、セクション/ラベル/重要度/習熟度での絞り込み。 |
| FR-V-08 | Excelインポート | テンプレート形式のExcel/CSVから単語を一括登録する（取込件数・スキップ件数を返却）。 |
| FR-V-09 | Excelエクスポート | 教材配下の単語をExcel出力する。 |
| FR-V-10 | テンプレート取得 | インポート用Excelテンプレートをダウンロードする。 |

#### 単語の項目

| 項目 | 必須 | 取りうる値 / 既定 |
|------|------|------|
| word（語） | ○ | 文字列（最大255） |
| meaning（意味） | ○ | 文字列（最大500） |
| partOfSpeech（品詞） | − | 文字列（例: 名/動/形/副）最大50 / null |
| importance（重要度） | − | 0 / 1 / 2（既定1） |
| label（ラベル） | − | easy / normal / hard（既定 normal） |
| proficiency（習熟度） | − | high / medium / low / null（既定 low、フォーム新規は未設定） |
| memo（メモ） | − | テキスト / null |
| exampleSentence（例文） | − | テキスト / null |
| exampleTranslation（和訳） | − | テキスト / null |
| exampleExplanation（例文説明） | − | テキスト / null |
| image（画像） | − | 画像ファイル / なし |
| sortOrder（並び順） | − | 整数（未指定時は末尾に自動採番） |

### 5.2 クイズ（FR-Q）

| ID | 機能 | 内容 |
|----|------|------|
| FR-Q-01 | 出題条件設定 | セクション・重要度・ラベルでの絞り込み、クイズタイプ、問題数、出題順序を設定する。 |
| FR-Q-02 | 出題単語の選定 | 間隔反復（SM-2）に基づき、復習期限・新規・苦手単語を優先配分して出題単語を選定する。 |
| FR-Q-03 | 4択クイズ | 単語に対し意味4択（うち正解1）を出題する。誤答選択肢は同一品詞を優先して生成。 |
| FR-Q-04 | 入力クイズ | 意味（日本語）を提示し、英単語のスペルを入力させる。大文字小文字は無視して採点。 |
| FR-Q-05 | 採点・正誤表示 | 回答の正誤を即時表示する。 |
| FR-Q-06 | 回答記録 | 各回答をバックエンドに記録し、SM-2で学習統計を更新する（クイズ進行を止めないバックグラウンド送信）。 |
| FR-Q-07 | 例文・画像表示 | 回答後に例文（対象語を強調）・和訳・例文説明・画像を表示する。 |
| FR-Q-08 | 習熟度のその場更新 | 回答画面から習熟度（高/中/低）を即時更新できる。 |
| FR-Q-09 | メモのその場編集 | 回答画面からメモを編集できる。 |
| FR-Q-10 | 前の問題へ戻る | 直前の問題に戻り、回答済みなら状態を復元表示する。 |
| FR-Q-11 | 結果表示 | 正答数/出題数、正答率、誤答単語一覧を表示する。 |
| FR-Q-12 | 誤答リトライ | 結果画面から、誤答した単語だけで再クイズを開始する。 |
| FR-Q-13 | 音声読み上げ | 英単語・例文をTTS（Web Speech API）で読み上げる（女性音声を優先選択）。 |

#### 出題条件（QuizSettings）

| 設定 | 取りうる値 / 既定 |
|------|------|
| sectionIds（対象セクション） | 複数選択（既定: 全選択） |
| importances（重要度絞り込み） | 0/1/2 の複数選択（既定: 指定なし=全件） |
| labels（ラベル絞り込み） | easy/normal/hard の複数選択（既定: 指定なし=全件） |
| quizType（クイズタイプ） | choice / input（既定 choice） |
| count（問題数） | 5〜（総単語数/上限50）、または全件（0扱い）（既定: 全件） |
| ordered（出題順序） | random（false）/ sequential（true）（既定 random） |
| vocabularyIds（特定単語指定） | リトライ・復習で利用（任意） |

### 5.3 フラッシュカード（FR-F）

| ID | 機能 | 内容 |
|----|------|------|
| FR-F-01 | 表示条件設定 | セクション・重要度・ラベル・単語数・出題順序、語表示時間・意味表示時間を設定する。 |
| FR-F-02 | 自動めくり表示 | 全画面で「語→意味」を自動で切り替え表示し、末尾到達で周回（周回数を表示）する。 |
| FR-F-03 | 一時停止/再開 | 画面タップまたはボタンで一時停止/再開できる。タブ非表示時は自動で一時停止。 |
| FR-F-04 | 音声読み上げ | 語の表示時にTTSで読み上げる。 |
| FR-F-05 | 進捗表示 | 「現在/総数（n周目）」とプログレスバーを表示する。 |

#### 表示時間設定

| 設定 | 範囲 / 既定 |
|------|------|
| wordDuration（語の表示時間） | 2〜10秒（既定5秒） |
| meaningDuration（意味の表示時間） | 2〜10秒（既定3秒） |

### 5.4 間違えた単語の復習（FR-R）

| ID | 機能 | 内容 |
|----|------|------|
| FR-R-01 | 期間指定 | 過去3日 / 過去7日 / 過去1ヵ月 / 任意日付（開始〜終了）から選択する。 |
| FR-R-02 | 誤答単語抽出 | 指定期間に誤答（is_correct=false）した単語を重複排除して抽出する。 |
| FR-R-03 | 復習クイズ開始 | 抽出した単語でクイズ（4択/入力）を開始する。 |
| FR-R-04 | 小テスト印刷 | 抽出した単語で小テストを印刷する。 |

### 5.5 小テスト印刷（FR-P）

| ID | 機能 | 内容 |
|----|------|------|
| FR-P-01 | 出力対象生成 | クイズ/復習の出題条件で対象単語を取得し、印刷用シートを生成する。 |
| FR-P-02 | テストタイプ選択 | meaning（意味を問う）/ spelling（スペルを問う）/ fill_spelling（例文穴埋め）から選択。 |
| FR-P-03 | 回答方式選択 | free（記述）/ choice（選択式）から選択。 |
| FR-P-04 | シート出力 | 問題用紙・回答用紙・解答の各シートを生成し、出力範囲（すべて/問題のみ/回答用紙のみ/解答のみ）を選択して `window.print()` で印刷する。 |
| FR-P-05 | 例文穴埋め制約 | fill_spelling では例文が登録された単語のみを対象とし、対象語を空欄化する。 |

### 5.6 進捗統計（FR-S）

| ID | 機能 | 内容 |
|----|------|------|
| FR-S-01 | 統計取得 | 教材単位で 総単語数 / 習得数 / 学習中数 / 新規数 / 全体正答率 / 復習期限到来数 を算出する。 |
| FR-S-02 | 習得判定 | repetition_count ≥ 3 かつ interval_days ≥ 7 を「習得」とみなす。 |

### 5.7 活動ウォッチ連携（FR-T）

| ID | 機能 | 内容 |
|----|------|------|
| FR-T-01 | 開始確認 | クイズ/フラッシュカード開始時にウォッチ未起動なら開始確認ダイアログを表示する。 |
| FR-T-02 | 計測開始 | ウォッチを `source=vocabulary`、`vocabularyType=quiz/flashcard`、`studyResourceId` 付きで開始する。 |
| FR-T-03 | スキップ可 | ウォッチを開始せず学習を進めることもできる。 |

## 6. 非機能要件

| 分類 | 要件 |
|------|------|
| 性能 | クイズ開始時に対象単語と4択を一括取得し、以降は端末内で進行する。回答記録は非同期送信し、クイズ進行をブロックしない。 |
| 可用性 | 回答記録の送信失敗時もサイレントに無視し、学習体験を維持する（記録のみ欠損）。 |
| 操作性 | スマホ/PC両対応のレスポンシブUI。入力クイズはEnterで送信、autofocus対応。 |
| アクセシビリティ | TTSによる発音支援（女性音声優先）。 |
| データ保全 | 単語・回答記録・学習統計は外部キー制約で整合性を担保。教材/セクション削除時はカスケード削除。単語削除時は画像も物理削除。 |
| セキュリティ | 全操作で教材所有者（user_id）を検証し、他ユーザーのデータへのアクセスを403で拒否。 |
| 永続化 | フロントは単語一覧・統計を localStorage に永続化（Piniaのpersist）。フォーム入力途中は復元可能。 |

## 7. データ要件（論理）

主要エンティティと関連は以下のとおり（詳細は詳細設計書を参照）。

```
StudyResource (1) ─< (N) StudyResourceSection (1) ─< (N) Vocabulary
                                                          │
                              ┌───────────────────────────┼───────────────────────────┐
                              │                            │
                  (N) VocabularyAttempt        (1) VocabularyLearningStat
                  ［ユーザー×単語の回答履歴］     ［ユーザー×単語の集約統計／SM-2］
```

- `Vocabulary` は `StudyResourceSection` に属する。
- `VocabularyAttempt` は回答1件ごとの履歴（user_id × vocabulary_id）。
- `VocabularyLearningStat` は user_id × vocabulary_id で一意の集約統計。

## 9. 制約・前提

- 単一ユーザー前提だが、データモデルは `user_id` でマルチユーザー対応可能。
- TTSはブラウザのWeb Speech API依存。利用可能な音声は環境により異なる。
- 小テスト印刷はブラウザの印刷機能（`window.print()`）に依存。
- SM-2の配分（復習60% / 新規30% / 苦手10%）・習得判定の閾値はサービス層にハードコードされている。
